The following image highlights the important parts of the Construct 3 user interface (UI) with numbers. Click the image to enlarge it. An overview of each part is provided below, and later the manual goes in to each section in more detail. Note that initially only the [Start Page](https://www.construct.net/make-games/manuals/construct-3/overview/start-page) is visible. Much of the interface will not appear until you create or open a project. Also note the exact appearance of Construct can depend on which theme you have selected.

![The Construct user interface](https://construct-static.com/images/v1721/uploads/articleuploadobject/0/images/139535/construct-user-interface.png)

1. [**Main menu button**](https://www.construct.net/make-games/manuals/construct-3/interface/main-menu): Click this to open the main menu. This provides options for basic tasks like opening and closing projects, exporting, changing settings and so on.
2. [**Main toolbar**](https://www.construct.net/make-games/manuals/construct-3/interface/main-toolbar): This provides shortcuts to the most commonly-used features: save, undo, redo and preview. Note the arrows next to some buttons which provide a dropdown menu with more options.
3. **View tabs:** These tabs let you switch the main view between different [layouts](https://www.construct.net/make-games/manuals/construct-3/project-primitives/layouts) (where you place objects) and [event sheets](https://www.construct.net/make-games/manuals/construct-3/project-primitives/events/event-sheets) (where you define logic using the event system). You can customise the color of these by right-clicking them and using the **Colors** menu.
4. **[Account](https://www.construct.net/make-games/manuals/construct-3/getting-started/using-an-account) badge**: This shows your account status. Click it to show the Account menu.
5. **Main view:** This is where the currently selected [Layout View](https://www.construct.net/make-games/manuals/construct-3/interface/layout-view) or [Event Sheet View](https://www.construct.net/make-games/manuals/construct-3/interface/event-sheet-view) appears. The view tabs select which is visible. In this picture, it's showing an empty Layout View.
6. [**Properties bar**](https://www.construct.net/make-games/manuals/construct-3/interface/bars/properties-bar): This lists all the properties for the selected item, allowing you to change settings for it.
7. [**Project bar**](https://www.construct.net/make-games/manuals/construct-3/interface/bars/project-bar): This lists everything in your project. It gives you an overview of what you've added, and lets you navigate around the project as well, such as by opening layouts or event sheets to view them.
8. [**Layers bar**](https://www.construct.net/make-games/manuals/construct-3/interface/bars/layers-bar): When a Layout View is open, this shows the layers on the layout.
9. **Tabs:** By default the Layers bar and [Tilemap bar](https://www.construct.net/make-games/manuals/construct-3/interface/bars/tilemap-bar) are docked together. You can use these tabs to switch between the bars. You can drag and drop bars around to rearrange them. You can dock or tab together any combination of bars you want to customise the interface.

## Pinning bars
You can click the pin icon in the caption of most bars to put them into an auto-hide mode. This displays a tab along the edge of the screen, and then when you hover the mouse cursor over the tab, the bar will re-appear and stay on-screen until you move the mouse cursor off the bar or click outside of the bar. This can be a convenient way to still easily access a bar without it always taking up space on screen.

## Changing the theme
You can choose a different theme to alter the appearance of Construct's interface, such as by choosing a dark theme. The current theme can be changed from [Settings](https://www.construct.net/make-games/manuals/construct-3/interface/dialogs/settings). Third-party themes can also be installed as [addons](https://www.construct.net/make-games/manuals/construct-3/tips-and-guides/installing-third-party-addons).
Note that the precise appearance of the interface depends on which theme you are using. The manual uses images based on the *Default (no theme)* style for a neutral appearance. If you change the theme, note the interface may look different to images in the manual.
